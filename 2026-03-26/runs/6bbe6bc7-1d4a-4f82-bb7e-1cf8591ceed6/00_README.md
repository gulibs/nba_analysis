# OpenClaw 分步执行说明

## 用法
按 **文件名数字顺序** 将每个 `.md` 作为**独立一轮对话的 user prompt**（或等价输入）依次交给 OpenClaw；**不要**一次性合并全部文件，以免上下文溢出。

## 建议工作流
1. 执行 `01` → 保留助手简短确认（规则已读）。
2. 执行 `02` → 得到赛程/场次完整性结论；若有 Fail Fast，停止。
3. 执行 `03` → 得到全量 `level_hint` / 缺失盘汇总表。
4. 对每个 `04_game_*.md` **分别执行**（一场一轮），得到单场结构化结论；场次多时可分批。
5. 将各场结论**摘要粘贴**进 `05` 再执行（或让 Claw 从会话历史读取，视产品能力而定）。
6. 执行 `06` 仅用于统一输出排版（可选）。
